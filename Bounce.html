Bounce World


Create a captivating gaming experience 'Bounce World' with improved visuals and exciting challenges. Can you craft a path to victory amidst the obstacles?


Welcome, developers! Your mission, should you choose to accept it, is to enhance the current baseline of the “Bounce World Game” web application, transforming it into a thrilling and interactive gaming experience. Building upon the existing HTML, CSS, and JavaScript code (Scaffold) provided, you will introduce new functionalities and improvements to uplift the gameplay and visuals to provide users with a captivating and immersive gaming environment.
Bounce World game is a game where there is one player and some steps, the player can either move in directions or can jump. Player should have to deal with enemies and hurdles. Player should be able to restrict itself from hurdles and enemies, if the player will be able to do so and will be able to end the game. In this situation, the player will win the game.


General Requirements

Your solution must maintain the foundational structure of the Scaffold, while bringing in functional upgrades and visually stimulating elements to improve the user experience.

Tasks for Students:

Core Requirements
  • Game Environment
  • Develop a vivid and dynamic backdrop using enchanting colors and visual elements to set the mood of the different levels in the game.
  • Design platforms and obstacles strategically to provide a challenging yet achievable gameplay experience.
  • Player Character
  • Enhance the player character design, ensuring smooth movement and transitions while jumping, running, and navigating through obstacles.
  • Obstacles
  • Introduce spikes and enemy patrols in the game, adding a layer of complexity and excitement.
  • Ensure that collision with spikes and enemies triggers an end game scenario, encouraging players to navigate carefully.
  • Design visually appealing and distinct graphics for the obstacles, players and enemies to differentiate them from other elements in the game.
  • Game Mechanics
  • You should display the game inside the canvas.
  • The game has to start when the 'Start Game' button is clicked.
  • Player can only move left or right using arrow keys.
  • Use the ‘space’ key to make the player jump.
  • If a player comes in contact with spikes or ‘hurdles’, it should stop jumping. In the video, we have described spikes with ‘red’ colour.
  • When a player collides with enemies, the game should end. In the video we have described the enemy from the moving ‘black’ box.
  • If a player falls off the screen, the game should be ended.

Bonus Features
  • Add creative animations to depict different states of the player character, such as running, jumping, and collision reactions.
  • Implement transition animations for changing levels and during the end game scenario to enhance the visual appeal.
  • Create a patrolling enemy with AI behavior, changing direction upon hitting the canvas edges to add unpredictability in the enemy movement.
